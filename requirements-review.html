Information Architecture

最终信息层级

Header1 和 Header3 是父子导航。Head 表达当前新闻的完整语义。Header2/Tabs 不只是导航， 它同时是默认的中部内容画面。

Header1 · 粗粒度栏目Intro / 要闻 / 开发生态 / 行业动态 / Outro，始终完整显示
Head · 新闻完整标题当前正在讲解的单条新闻标题
Header2 / Tabs · 内容基础层默认采用 3 列 × 最多 2 行的大卡片矩阵，当前重点可高亮

Playback Rules

已确认的播放规则

1

三级状态同步

当前 Story 决定 Header1 所属栏目、Head 完整标题，以及 Header3 当前新闻。

2

Active Tab 随场景变化

同一条新闻播放期间，不同 Scene 可以高亮不同 Tab，帮助观众快速抓住当前重点。

3

真实图片覆盖 Tabs

图片在 Tabs 内容容器内部上下左右居中，高度占容器 92%、最大宽度 96%；容器边界保证图片不会碰到 Header3。

4

JSON 描述内容与状态

JSON 维护栏目、新闻、Tabs、Scene、Active Tab、图片覆盖、字幕与持续时间。

5

短促克制的动效

高亮状态平滑切换；真实图片使用短淡入淡出和轻微缩放，不追求复杂炫技。

6

导航宽度表达时长

Header1 与 Header3 每一项的宽度，按该项实际播放时长占同层总时长的比例自动分配。

7

网页与视频共用模型

JSON 稳定后，HTML 可视化编辑器与 Remotion 视频渲染器共享同一数据模型和组件规则。

8

Tab 支持受限 Markdown

Tab 内容仅解析行内代码和粗体，用于标注术语与重点；不支持会破坏卡片布局的任意 Markdown。

9

字幕采用宽幅渐变浮层

字幕固定横跨 Tabs 容器底部，使用大号白字、文字阴影和由透明到深色的渐变背景；允许遮挡图片最下方少量区域。

10

Tabs 使用紧凑大卡片矩阵

最多 6 张时采用固定 3 列 × 2 行布局，卡片等宽等高、窄且一致的间距、内部左对齐，并尽量铺满内容容器。

Proposed JSON Contract

建议的稳定 JSON 结构

Tabs 只定义卡片内容；Scenes 定义时间线上当前高亮哪张卡片、是否出现真实图片以及字幕。 这样不会重复维护相同的卡片文案。

为什么 Active 放在 Scene？

因为 Active 是播放时状态。同一组 Tabs 在不同时间可以高亮不同卡片。

为什么 Overlay 放在 Scene？

真实图片只在特定时间出现，并且用于证明当前正在讲解的重点。

Header 宽度如何得到？

程序先汇总 Scene、Story 和栏目时长，再用“当前项时长 ÷ 同层总时长”计算 Header1 与 Header3 的宽度占比。

为什么不生成任意 HTML？

优先用稳定组件渲染 JSON，才能保证每日内容变化后仍然不会溢出或破坏画面。

Markdown 支持到什么程度？

只支持 `行内代码` 与 **粗体**。渲染前解析为受控 React 节点，不直接注入任意 HTML。

后续扩展点

Overlay 可继续增加裁剪区域、图片缩放、标注框和局部放大，但不影响基础 JSON 模型。

Human Review

请重点审查这些事项

以下事项已经完成本轮人工审查，可作为首版 Remotion 模板的明确实现基线。

结构与内容规则

  • Header1 是粗粒度栏目导航，Header3 是细粒度新闻导航，二者始终完整显示。
  • Header1 与 Header3 的项目宽度按实际播放时长比例自动分配。
  • Head 位于 Header1 与 Tabs 之间，显示当前新闻完整标题。
  • Tabs 使用居中 Grid 排列，每页最多显示 6 张，并允许当前重点高亮。
  • 最多 6 张 Tabs 默认采用固定 3 列 × 2 行矩阵，卡片等宽等高并以窄间距铺满内容容器。
  • Tab 内容支持受限 Markdown：行内代码背景框与粗体。
  • Active Tab 根据 Scene 变化，不是整条新闻固定状态。
  • 真实图片覆盖在 Tabs 上方，Tabs 在背景中压暗保留。
  • 真实图片相对 Tabs 内容容器居中，高度占该容器的 92%，最大宽度为 96%，不会越过容器碰到 Header3。
  • 内容、时间线、图片和字幕最终由一份 JSON 驱动。

视觉与播放基线

  • 真实图片高度占 Tabs 内容容器的 92%，宽度按图片比例自适应，最大不超过容器的 96%。
  • 图片覆盖期间，底层 Tabs 保留 18% 可见度。
  • 字幕始终按整句显示，不做逐词高亮。
  • 字幕横跨 Tabs 内容容器底部，使用大号白字、文字阴影和深色渐变底，允许轻微遮挡图片底边。
  • 日报包含固定结构的 Intro 和 Outro，但不加入学习复盘或明日行动内容。
  • Intro 用于日期、栏目和今日要点预告；Outro 用于简短结束语。
  • Intro 与 Outro 同样由字幕/TTS 决定时长，并参与 Header 宽度配额。
  • Header1、Head 与 Header3 不被真实图片覆盖，始终保持上下文可见。
