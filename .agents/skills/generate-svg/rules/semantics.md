# Semantic Selection Rules

Choose the icon from the tab's meaning, not merely the first keyword in its title.

## Selection Process

1. Read the tab title and summary together.
2. Identify the tab's central noun, action, or consequence.
3. Compare sibling tabs and choose visually distinct silhouettes.
4. Prefer a familiar symbol that remains understandable without the title.
5. Use an emoji prefix as a strong hint, not an obligation to copy its exact appearance.

## Common Concepts

Use these as starting points, not a rigid lookup table:

| Meaning | Possible concepts |
| --- | --- |
| safety, stability, protection | shield, check, lock |
| developer, code, API | code brackets, terminal, connected blocks |
| agent, automation | robot, nodes, branching hierarchy |
| performance, speed | lightning, gauge, rising line |
| testing, beta, experiment | flask, test tube |
| reasoning, intelligence | brain, connected thought nodes |
| impact, change, tradeoff | ripple, diverging arrows, warning |
| future, roadmap | route, upward path, flag |
| launch, release | rocket, outward arrow |
| news, overview | newspaper, grouped cards |

When several meanings apply, choose the concept that best distinguishes the tab from its siblings.
Do not use generic stars or rockets when a clearer semantic symbol exists.

## Color Selection

- Choose a vivid color that fits the concept and follows [theme.md](theme.md).
- Use sibling color variety to help scanning, but do not assign colors mechanically by index.
- Use semantic associations when useful: yellow for warning or energy, cyan for technology, green for
  success, coral for change, and purple for intelligence.
- It is acceptable for an icon to use one, two, or several colors. Every color must have a visual job.
