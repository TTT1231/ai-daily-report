Quick start

你想做什么？

选择目标，直接跳到对应的配置位置。

添加 LinuxDo RSS 增加新的栏目或 RSS 2.0 来源 关注新品牌或产品 添加重点实体及其常见名称 添加直接加分词 命中任意标题后获得 9 分保底 排除不喜欢的内容 添加硬排除词或自然语言偏好

网络代理按来源决定：sources.jsonc 里标了 "proxy": true 的来源（如 linux.do，在 Cloudflare 后面）抓取时走 .env 的小写 all_proxy=http://127.0.0.1:7890，其他来源直连、不被代理拖累； AI 模型请求仍受 all_proxy 控制（配置后即走）。标了 proxy: true 的来源若 all_proxy 未配或无效会报错，不会静默回退直连。采集器只读小写 all_proxy， 不读其他代理变量，也不自动探测本地代理端口。

Configuration

日常只维护两个文件

来源配置与兴趣偏好职责分离，运行状态由程序自动维护。

文件 用途 维护方式
sources.jsonc RSS 来源、启用状态与分页参数 人工维护 preferences.jsonc 重点实体、加分词、排除词与阈值 人工维护 rss-state.json 最近一次完整抓取记录，用于补选与失败来源保留 自动生成

两个 JSONC 文件都带字段注释，可以直接编辑。AI 评分 Prompt 与本地关键词规则共享同一份 preferences.jsonc。

sources.jsonc

添加自己的 RSS 来源

新增 LinuxDo 栏目时，复制现有对象并修改 id、name 和 url。

LinuxDo 来源示例 sources.jsonc 
{
  "id": "linuxdo-new-category",
  "name": "Linux.do 新栏目",
  "adapter": "linuxdo",
  "url": "https://linux.do/c/example/123.rss",
  "enabled": true,
  "maxPages": 2,
  "pageStart": 0,
  "pageDelaySeconds": 10
}
id
来源的唯一稳定标识。修改后会被视为一个新来源。
adapter: "linuxdo"
自动分页，并使用 Topic ID 跨栏目去重。
adapter: "rss2"
用于普通 RSS 2.0，且 maxPages 必须为 1。
enabled
设为 false 可临时停用来源。
priorityEntities

添加重点品牌或产品

name 用于告诉 AI 关注对象是谁；aliases 是标题中可能出现的名称。

添加 Google Gemini preferences.jsonc 
{
  "name": "Google Gemini",
  "aliases": ["google gemini", "gemini", "谷歌双子座"]
}

标题命中重点实体后会通过本地兴趣门槛；同时命中 importantChanges 时获得 9 分保底。

signals

添加加分关键词

不同信号词承担不同职责，最直接的入口是 priorityKeywords。

推荐 signals.priorityKeywords

任意标题直接加分

命中后获得 9 分本地保底，适合 MCP、特定项目或长期关注主题。

"priorityKeywords": [
  "mcp",
  "你自己的直接加分词"
]
组合条件 signals.importantChanges

重点实体变化词

重点实体同时命中发布、开源、价格等变化词时，获得 9 分保底。

"importantChanges": [
  "发布", "开源", "上线",
  "你关注的变化词"
]
riskEvents封号、跑路、关停等风险事件
majorPolicy国家级 AI 政策与监管信号
majorIndustryChanges非重点实体的行业级变化
dislikes & guidance

排除与不喜欢的内容

明确关键词交给本地规则，复杂语义交给 AI 理解。

硬排除 dislikes.hardExclude

命中后无条件排除

"hardExclude": [
  "招聘",
  "你明确不想看的词"
]
AI 语义 guidance.notInterested

描述难以关键词化的偏好

"notInterested": [
  "不关注普通软件的小版本更新"
]
thresholds

调整筛选严格程度

默认配置适合日常日报；提高最低分会让内容更精简。

默认最低分 7 minimumScore
候选数量上限 30 maximumCandidates

minimumScore 越高越严格。通常保持 7 即可，设置过高可能遗漏有价值但影响有限的内容。

Pipeline

运行流程

从配置读取到生成日报，共经过五个阶段。

  1. 01
    读取配置sources.jsonc + preferences.jsonc
  2. 02
    抓取与解析sources.go → rss2.go
  3. 03
    快照与兴趣筛选state.go → ranking.go
  4. 04
    聚类与视频编排grouping.go → story_tabs.go
  5. 05
    生成日报数据data-scheme/data.json
保存配置后运行 bun run rss
bun run rss
Runtime state

rss-state.json 是什么？

它是自动生成的运行状态，不是用户配置。

保存最近一次完整抓取记录

用于人工补选新闻和来源失败时保留上次状态；评分仍会处理最近时间窗口内的全部条目。

删除后只会丢弃快照；下一次运行会重新抓取并自动重建。
来源失败时保留该来源上一次运行状态，避免短暂故障清空快照。
跨栏目时LinuxDo 同一 Topic 只处理一次。
Directory

目录职责

配置、处理逻辑和自动生成状态各自独立。

ingest/项目目录
ingest/
├── sources.jsonc           # 人工维护：RSS 来源
├── preferences.jsonc       # 人工维护：用户兴趣画像
├── sources.go              # 来源校验、多来源抓取与跨来源去重
├── preferences.go          # 兴趣画像校验与评分 Prompt 生成
├── linuxdo_adapter.go      # LinuxDo 分页与 Topic ID
├── rss2.go                 # 通用 RSS 2.0 下载与解析
├── vpnproxy.go             # 按来源 all_proxy 策略（sources.jsonc 的 proxy 字段）
├── state.go                # 快照读写与失败来源保留
├── ranking.go              # 本地评分与组合排除规则
├── grouping.go             # 合并相似新闻
├── story_tabs.go           # 生成视频 Tabs 和字幕
├── rss-state.json          # 自动生成的运行状态
├── readme.html             # 当前说明页
└── readme.css              # 当前说明页样式
